# 업무 AI 에이전트 설계 Flow 연결 가이드

이 flow는 업무 설명을 입력받아 AI 에이전트 설계서를 만들어주는 체험용 Langflow flow입니다.

처음에는 LLM 없이 기본 결과를 확인하고, 이후 프롬프트 템플릿과 LLM을 연결하는 방식으로 확장하는 것을 권장합니다.

## 1. 컴포넌트 등록

Langflow 커스텀 컴포넌트 경로에 아래 폴더를 추가합니다.

```text
C:\Users\qkekt\Desktop\기능flow\business_agent_design_flow\langflow_components\business_agent_design_flow
```

Langflow에서 컴포넌트가 보이지 않으면 서버 또는 화면을 새로고침합니다.

| 2 | 01 업무 프로세스 구조화 | 업무 구조화 결과 | 02 AI 에이전트 기능 카탈로그 | 업무 구조화 결과 |
| 3 | 02 AI 에이전트 기능 카탈로그 | 기능 카탈로그 결과 | 04 AI 에이전트 설계 결과 정리 | 기능 카탈로그 결과 |

## 4. 00 업무 설명 입력에 넣는 값

### 업무 설명

필수 입력입니다. 사람이 실제로 하는 업무를 자연어로 적습니다.

예시:

```text
매일 아침 설비별 생산량과 불량률을 엑셀로 내려받아 전일 대비 급증한 설비를 확인합니다.
이상 설비가 있으면 원인 후보를 정리해서 담당자에게 메일로 공유하고, 긴급 건은 팀장 승인 후 후속 조치를 등록합니다.
```

### 업무 목적/대상

선택 입력입니다. 결과를 누가 보고 무엇을 결정하는지 적습니다.

### 사용 데이터/시스템

선택 입력입니다. Excel, DB, API, 사내 시스템, 메일, 문서 등 사용하는 데이터를 적습니다.

### 제약사항

선택 입력입니다. 보안, 승인, 사람이 꼭 봐야 하는 단계, 자동화하면 안 되는 작업을 적습니다.

### 원하는 결과물

선택 입력입니다. 리포트, 알림, 메일 초안, 처리 체크리스트, 대시보드 등 원하는 산출물을 적습니다.

## 5. 03 추가 설계 지시사항에 넣으면 좋은 내용

LLM 연결 흐름에서만 사용됩니다.

예시:

```text
초보 Langflow 개발자가 따라 만들 수 있게 노드 수를 너무 늘리지 말고 1차/2차 구현으로 나눠줘.
사내 시스템 업데이트는 실제 자동 실행하지 말고 승인 후 실행 후보로만 제안해줘.
기존 기능flow 중 데이터 조회가 필요하면 reusable_data_flow를, 결과 리포트가 필요하면 html_report_flow를 추천해줘.
```

## 6. 02 추가 기능 간단 입력에 넣는 방법

기본 기능 외에 사내에서 사용할 수 있는 기능이나 이미 만든 flow가 있으면 `02 AI 에이전트 기능 카탈로그`의 `추가 기능 간단 입력`에 적습니다.

한 줄에 기능 하나씩 아래 형식으로 입력합니다.

```text
기능명 | 설명 | 사용 상황 | 필요 입력 | 산출물 | 난이도 | 구현 힌트 | 참고 링크
```

예시:

```text
사내 티켓 조회 | 티켓 상태와 담당자를 조회합니다. | 업무가 티켓 상태 확인이나 처리 이력 조회를 포함할 때 | 티켓 ID, 요청자 | 티켓 상태, 담당자, 최근 처리 이력 | 중급 | 처음에는 조회 전용으로 연결하고 등록은 사람 승인 뒤 진행합니다. | 사내 티켓 API 문서 링크
메일 초안 생성 | 업무 결과를 담당자에게 보낼 메일 초안으로 정리합니다. | 결과 공유나 후속 조치 요청이 필요한 업무일 때 | 수신자, 핵심 요약, 조치 요청 | 메일 제목, 메일 본문 초안 | 초급 | 자동 발송하지 말고 초안만 출력하도록 설계합니다. |
```

필수로 잘 적어야 하는 값은 `기능명`, `설명`, `사용 상황`입니다. 뒤쪽 값은 비워도 기본값으로 보완됩니다.

JSON에 익숙한 사용자는 `추가 기능 카탈로그 JSON (고급)`에 직접 구조를 넣어도 됩니다.

## 8. 기존 기능flow와 연결하는 기준

이 flow는 실제 자동화를 바로 실행하지 않고, 어떤 flow를 붙이면 좋은지 설계합니다.

- 데이터 조회가 핵심이면 `reusable_data_flow`를 추천합니다.
- 조회 결과를 리포트로 보여줘야 하면 `html_report_flow`를 추천합니다.
- 외부 시스템을 여러 개 조작해야 하면 `에이전트와 도구` 또는 `MCP 도구`를 2차 확장안으로 제안합니다.
- 승인, 고객 발송, 시스템 업데이트는 `human_review_gate` 패턴을 우선 제안합니다.
