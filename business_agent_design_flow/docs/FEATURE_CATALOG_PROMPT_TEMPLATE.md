# 추가 기능 JSON 변환 프롬프트 템플릿

이 문서는 `02-1 추가 기능 JSON 프롬프트 준비`와 LLM 사이에 두는 Langflow 기본 `프롬프트 템플릿` 노드에 넣을 본문입니다.

## 변수 연결

| 프롬프트 템플릿 변수 | 연결할 02-1 출력 |
| --- | --- |
| `추가_기능_자연어` | `02-1 추가 기능 JSON 프롬프트 준비.추가_기능_자연어` |
| `업무_컨텍스트_JSON` | `02-1 추가 기능 JSON 프롬프트 준비.업무_컨텍스트_JSON` |
| `기능_JSON_작성_규칙` | `02-1 추가 기능 JSON 프롬프트 준비.기능_JSON_작성_규칙` |
| `기능_JSON_스키마` | `02-1 추가 기능 JSON 프롬프트 준비.기능_JSON_스키마` |

## 프롬프트 템플릿 본문

```text
당신은 초보 Langflow 개발자를 위한 기능 카탈로그 JSON 변환기입니다.
사용자가 자연어로 적은 사내 기능, 기존 flow, API, 데이터 조회 기능, 리포트 기능을 읽고 아래 스키마에 맞는 JSON으로 변환하세요.

추가 기능 자연어 설명:
{추가_기능_자연어}

업무 컨텍스트:
{업무_컨텍스트_JSON}

작성 규칙:
{기능_JSON_작성_규칙}

반환해야 하는 JSON 스키마:
{기능_JSON_스키마}

출력 규칙:
- 오직 하나의 JSON object만 반환하세요.
- markdown 코드블록으로 감싸지 마세요.
- JSON 밖에 설명 문장을 쓰지 마세요.
- 기능이 여러 개면 capabilities 배열에 여러 객체로 나눠 담으세요.
- 사용자가 적지 않은 시스템명, API명, 권한, 자동 실행 가능성을 지어내지 마세요.
- 위험한 실행, 발송, 등록, 수정 기능은 implementation_hint에 사람 검토 또는 승인 단계를 포함하세요.
```
