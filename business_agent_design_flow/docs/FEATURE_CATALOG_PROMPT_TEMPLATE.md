# 추가 기능 JSON 변환 프롬프트 템플릿

이 문서는 이전 버전의 참고용 문서입니다.

현재 기본 flow에서는 추가 기능을 JSON으로 따로 만들지 않습니다. 사용자는 `00 업무 설명 입력`의 `업무 설명` 한 칸에 아래처럼 자연어로 적으면 됩니다.

```text
우리 팀에는 사내 티켓 조회 API가 있습니다.
티켓 ID를 넣으면 티켓 상태, 담당자, 최근 처리 이력을 조회할 수 있습니다.
처음에는 조회 전용으로만 쓰고, 티켓 수정이나 등록은 사람 승인 뒤에만 하도록 설계해주세요.
```

그러면 `02 AI 에이전트 기능 카탈로그` 노드가 업무 설명에서 추가 기능 후보를 자동으로 추론해 `user_added_*` capability로 붙입니다.

이전 방식의 `02-1 추가 기능 JSON 프롬프트 준비`, `02-2 추가 기능 JSON 정리` 노드는 `legacy_components` 폴더에 보존되어 있습니다. 일반 사용자는 연결하지 않아도 됩니다.
