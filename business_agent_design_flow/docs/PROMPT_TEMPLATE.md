# 업무 AI 에이전트 설계 Flow 프롬프트 안내

현재 기본 방식에서는 별도 `프롬프트 템플릿` 노드를 만들 필요가 없습니다.

`03 AI 에이전트 설계 프롬프트 준비` 노드가 아래 정보를 모두 포함한 완성 프롬프트를 만들고, 이 출력을 LLM의 `input`에 바로 연결합니다.

- 사용자 업무 요청
- 업무 프로세스 구조화 결과
- 기본 AI 에이전트 기능 카탈로그
- 사용자가 업무 설명 안에 적은 추가 기능 후보
- 작성 규칙
- 출력 JSON 스키마

## 기본 연결

| 순서 | From | Output | To | Input |
| --- | --- | --- | --- | --- |
| 1 | 02 AI 에이전트 기능 카탈로그 | 기능 카탈로그 결과 | 03 AI 에이전트 설계 프롬프트 준비 | 기능 카탈로그 결과 |
| 2 | 03 AI 에이전트 설계 프롬프트 준비 | LLM 설계 프롬프트 | LLM | input |

## 프롬프트 템플릿 노드를 꼭 쓰고 싶을 때

팀 표준상 Langflow 기본 `프롬프트 템플릿` 노드를 반드시 거쳐야 한다면, 변수는 하나만 사용하세요.

| 프롬프트 템플릿 변수 | 넣을 값 |
| --- | --- |
| `설계_컨텍스트_JSON` | `03 AI 에이전트 설계 프롬프트 준비`가 만든 전체 설계 컨텍스트 |

템플릿 본문 예시는 아래처럼 단순하게 유지합니다.

```text
당신은 초보 Langflow 개발자를 위한 업무 AI 에이전트 설계 코치입니다.
아래 설계 컨텍스트를 바탕으로 업무 프로세스 로직과 AI 에이전트화 개선 아이디어를 JSON으로 작성하세요.

설계 컨텍스트:
{설계_컨텍스트_JSON}

반드시 지켜야 할 규칙:
- 오직 하나의 JSON object만 반환하세요.
- markdown 코드블록으로 감싸지 마세요.
- JSON 밖에 설명 문장을 쓰지 마세요.
- suggested_capabilities에는 컨텍스트의 기능 카탈로그에 있는 capability_id만 사용하세요.
- 초보자가 바로 따라 만들 수 있게 Langflow 노드 연결 순서와 검증 방법을 구체적으로 적으세요.
- 위험한 작업은 자동 실행보다 사람 검토 또는 승인 Gate를 제안하세요.
```

다만 현재 구현에서는 이 템플릿을 따로 연결하지 않아도 됩니다. 가장 단순한 방식은 `03 -> LLM -> 04`입니다.
