# 업무 AI Agent 설계 Flow 샘플 입력

이 파일은 새 `business_agent_design_flow`를 테스트할 때 바로 복사해서 쓸 수 있는 샘플입니다.

## Case 1. 생산/품질 위험 설비 점검

### 00 업무 설명 입력

```text
매일 아침 생산 실적과 불량 데이터를 엑셀로 내려받아 설비별 이상 징후를 확인합니다.
불량률이 전일 대비 급증한 설비는 최근 작업 이력과 정비 이력을 같이 조회해서 원인 후보를 정리합니다.
결과는 아침 회의 전에 팀장에게 공유하고, 긴급 건은 담당자에게 메일 초안을 작성합니다.
메일은 자동 발송하지 말고 사람이 확인한 뒤 보내야 합니다.
사내 설비 이력 API가 있어서 설비 ID와 기준 일자를 넣으면 최근 작업 이력과 정비 이력을 조회할 수 있습니다.
```

확인 포인트:

- `reusable_data_flow`, `html_report_flow`, `human_review_gate`가 추천되는지
- 메일 자동 발송 금지가 사람 검토 지점으로 반영되는지
- 현재 Flow와 개선 Flow가 HTML에 모두 표시되는지

## Case 2. 티켓 분류와 답변 초안

### 00 업무 설명 입력

```text
고객 문의 티켓이 들어오면 담당자가 내용을 읽고 장애, 사용 문의, 권한 요청, 데이터 정정 요청으로 분류합니다.
장애나 권한 요청은 담당 부서에 배정하고, 단순 사용 문의는 답변 초안을 만들어 고객에게 공유합니다.
고객에게 보내는 답변은 반드시 사람이 확인한 뒤 발송해야 하고, 권한 변경은 승인자 확인 뒤에만 처리해야 합니다.
티켓 시스템 API로 티켓 상태와 과거 처리 이력을 조회할 수 있습니다.
```

확인 포인트:

- 분류-추천-사람 검토 패턴이 추천되는지
- 고객 발송과 권한 변경이 human review로 잡히는지
- API 연동이 미검증 제안 또는 API Request 후보로 표시되는지

## Case 3. 카탈로그 등록 Flow 테스트

### 2.1 카탈로그 원문 입력

```text
사내 티켓 조회 API는 티켓 ID를 입력하면 티켓 상태, 담당자, 최근 처리 이력을 조회할 수 있다.
티켓 수정이나 담당자 변경은 사람 승인 뒤에만 해야 한다.
고객 문의 분류, 장애 대응, 권한 요청 처리 업무에서 사용할 수 있다.

메일 초안 생성 기능은 분석 결과나 티켓 처리 결과를 바탕으로 제목과 본문 초안을 만든다.
실제 발송은 하지 않고 사람이 확인한 뒤 발송해야 한다.
```

확인 포인트:

- `2.3 카탈로그 JSON 검증`에서 2개 이상의 항목이 생성되는지
- 발송/수정/승인 관련 항목의 `human_review_required`가 `true`가 되는지
- Mongo URI가 없을 때 `23`이 skipped 상태와 preview를 반환하는지
