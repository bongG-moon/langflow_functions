# 업무 AI 에이전트 설계 Flow 한 파일 테스트 입력 모음

이 문서는 Langflow에서 바로 복사해 테스트할 수 있는 입력 예시입니다.

## 사용 방법

1. `00 업무 설명 입력`의 각 입력칸에 값을 복사합니다.
2. LLM 없이 확인할 때는 `00 -> 01 -> 02 -> 04 -> 05 -> Chat Output`으로 연결합니다.
3. LLM을 연결할 때는 `03 AI 에이전트 설계 프롬프트 준비`의 출력들을 프롬프트 템플릿 변수에 연결합니다.
4. `03 추가 설계 지시사항`은 LLM 연결 흐름에서만 사용합니다.

## 테스트 1. 생산/품질 일일 점검 업무

### 00.업무 설명

```text
매일 아침 설비별 생산량과 불량률을 엑셀로 내려받아 전일 대비 급증한 설비를 확인합니다.
이상 설비가 있으면 생산량, 불량률, 최근 작업 이력을 같이 보고 원인 후보를 정리합니다.
정리한 내용은 담당자에게 메일로 공유하고, 긴급 건은 팀장 승인 후 후속 조치를 사내 시스템에 등록합니다.
```

### 00.업무 목적/대상

```text
생산/품질 담당자가 아침 회의 전에 위험 설비를 빠르게 파악하고 후속 조치 우선순위를 정하는 것이 목적입니다.
```

### 00.사용 데이터/시스템

```text
Excel 생산 실적 파일, 품질 불량률 파일, 사내 설비 이력 시스템, 메일
```

### 00.제약사항

```text
사내 시스템 등록은 팀장 승인 이후에만 가능해야 합니다.
메일 발송은 자동 발송이 아니라 초안 작성 후 사람이 확인하고 보내야 합니다.
```

### 00.원하는 결과물

```text
위험 설비 요약, 원인 후보, 담당자 공유 메일 초안, Langflow 구현 순서
```

### 03.추가 설계 지시사항

```text
초보 Langflow 개발자가 이해할 수 있게 1차 구현은 실제 시스템 등록 없이 리포트와 메일 초안 생성까지만 제안해줘.
2차 구현에서 reusable_data_flow와 html_report_flow를 연결하는 확장안을 제시해줘.
```

### 02.추가 기능 간단 입력

```text
설비 이력 조회 | 설비별 최근 작업 이력과 정비 이력을 조회합니다. | 업무가 설비 이상 원인 확인을 포함할 때 | 설비 ID, 기준 일자 | 최근 작업 이력, 정비 이력, 이상 이력 | 중급 | 처음에는 조회 전용으로 연결하고 수정/등록은 사람 승인 뒤 진행합니다. |
담당자 메일 초안 생성 | 위험 설비 요약을 담당자에게 보낼 메일 초안으로 정리합니다. | 분석 결과를 담당자에게 공유해야 할 때 | 담당자, 위험 설비 목록, 원인 후보 | 메일 제목, 메일 본문 초안 | 초급 | 자동 발송하지 말고 초안만 출력합니다. |
```

## 테스트 2. 고객 VOC 분류 및 답변 초안 업무

### 00.업무 설명

```text
고객센터에서 접수된 VOC를 매일 확인하고, 문의 유형과 긴급도를 분류합니다.
반품, 장애, 계약 문의는 담당 부서별로 나누고, 긴급 고객은 우선 처리 대상으로 표시합니다.
담당자는 과거 유사 답변을 참고해 답변 초안을 만들고, 민감한 고객이나 보상 관련 건은 팀장 검토 후 답변합니다.
```

### 00.업무 목적/대상

```text
고객센터 담당자가 VOC를 빠르게 분류하고 답변 초안 품질을 일정하게 유지하는 것이 목적입니다.
```

### 00.사용 데이터/시스템

```text
VOC 접수 목록, 고객 등급 정보, 과거 답변 문서, 담당 부서 매핑표
```

### 00.제약사항

```text
고객에게 답변을 자동 발송하면 안 됩니다.
보상, 계약, 민감 고객 관련 답변은 반드시 팀장 검토가 필요합니다.
```

### 00.원하는 결과물

```text
VOC 분류표, 우선 처리 목록, 답변 초안, 검토 필요 목록
```

### 03.추가 설계 지시사항

```text
AI 에이전트가 직접 답변을 보내는 구조는 제외하고, 답변 초안과 검토 체크리스트를 만드는 구조로 제안해줘.
MCP나 외부 도구 연결은 2차 확장안으로만 언급해줘.
```

### 02.추가 기능 간단 입력

```text
유사 답변 검색 | 과거 답변 문서에서 유사한 VOC 답변을 찾습니다. | 고객 문의 답변 초안 작성이 필요한 업무일 때 | VOC 내용, 문의 유형 | 유사 답변 후보, 참고 문구 | 중급 | 처음에는 검색 결과만 제공하고 최종 답변은 사람이 검토합니다. |
민감 고객 검토 체크리스트 | 보상, 계약, 민감 고객 건을 검토 목록으로 정리합니다. | 고객 답변 전에 승인 여부를 확인해야 할 때 | 고객 등급, 문의 유형, 보상 여부 | 검토 필요 여부, 체크리스트 | 초급 | 위험 항목은 항상 사람 검토 단계로 넘깁니다. |
```

## 테스트 3. 월간 비용 마감 검증 업무

### 00.업무 설명

```text
월말마다 부서별 비용 데이터를 ERP에서 내려받고, 예산 대비 초과 항목을 확인합니다.
초과 항목은 담당 부서에 사유를 요청하고, 회신 내용을 취합해 마감 보고서를 만듭니다.
비용 코드가 잘못 들어간 경우에는 수정 요청을 등록하고, 확정 전에는 재무팀장이 최종 검토합니다.
```

### 00.업무 목적/대상

```text
재무 담당자가 비용 초과와 코드 오류를 빠르게 확인하고 마감 보고서를 안정적으로 만들기 위한 업무입니다.
```

### 00.사용 데이터/시스템

```text
ERP 비용 데이터, 예산 기준표, 부서 담당자 목록, 이메일, 마감 보고서 양식
```

### 00.제약사항

```text
ERP 데이터 수정은 자동으로 수행하면 안 되고, 수정 요청 초안까지만 만들어야 합니다.
최종 마감 보고서는 재무팀장 검토가 필요합니다.
```

### 00.원하는 결과물

```text
초과 비용 목록, 부서별 사유 요청 메일 초안, 코드 오류 후보, 마감 보고서 초안
```

### 03.추가 설계 지시사항

```text
데이터 조회와 리포트 생성은 기존 기능flow를 재사용하는 방향으로 제안해줘.
초급자가 먼저 만들 수 있는 최소 flow와 이후 확장 flow를 구분해서 설명해줘.
```

### 02.추가 기능 간단 입력

```text
ERP 비용 조회 | ERP에서 부서별 비용과 비용 코드를 조회합니다. | 월간 비용 마감 검증 업무일 때 | 마감월, 부서, 비용 코드 | 비용 목록, 예산 대비 차이, 비용 코드 | 중급 | 조회 전용으로 시작하고 ERP 수정은 자동화하지 않습니다. |
부서 사유 요청 초안 | 초과 비용 항목에 대해 부서 담당자에게 보낼 요청 메일을 작성합니다. | 예산 초과 사유를 취합해야 할 때 | 부서명, 초과 항목, 담당자 | 사유 요청 메일 초안 | 초급 | 담당자가 최종 문구를 확인한 뒤 발송합니다. |
```

## 테스트 4. 주간 프로젝트 리스크 점검 업무

### 00.업무 설명

```text
매주 프로젝트별 일정, 이슈, 담당자 코멘트를 취합해서 지연 위험을 확인합니다.
일정이 3일 이상 밀리거나 blocker 이슈가 있는 프로젝트는 위험으로 표시합니다.
PM에게 후속 조치 요청 메시지를 보내고, 임원 보고용으로 주간 리스크 요약을 작성합니다.
```

### 00.업무 목적/대상

```text
PMO 담당자가 여러 프로젝트의 지연 위험을 빠르게 파악하고 임원 보고 자료를 준비하는 것이 목적입니다.
```

### 00.사용 데이터/시스템

```text
프로젝트 일정표, 이슈 관리 시스템, PM 코멘트, 주간 보고서 양식
```

### 00.제약사항

```text
임원 보고 내용은 자동 확정하면 안 되고 담당자가 최종 문구를 검토해야 합니다.
PM에게 보내는 메시지도 초안으로만 생성해야 합니다.
```

### 00.원하는 결과물

```text
위험 프로젝트 목록, 후속 조치 요청 초안, 임원 보고 요약, Langflow 구현 로드맵
```

### 03.추가 설계 지시사항

```text
사용자가 보기 좋은 구조로 카드 요약과 프로세스 표가 꼭 포함되게 제안해줘.
리스크 판단 기준은 일정 3일 이상 지연 또는 blocker 이슈 존재로 명확히 반영해줘.
```

### 02.추가 기능 간단 입력

```text
이슈 관리 시스템 조회 | 프로젝트별 blocker 이슈와 담당자 코멘트를 조회합니다. | 프로젝트 지연 위험 판단에 이슈 정보가 필요한 업무일 때 | 프로젝트 ID, 기준 주차 | blocker 이슈, 담당자, 최신 코멘트 | 중급 | 처음에는 조회 결과만 가져오고 이슈 상태 변경은 하지 않습니다. |
임원 보고 요약 생성 | 위험 프로젝트와 주요 원인을 임원 보고용 문장으로 요약합니다. | 주간 리스크 보고 자료를 작성해야 할 때 | 위험 프로젝트 목록, 지연 사유, 후속 조치 | 임원 보고 요약, 핵심 리스크 문장 | 초급 | 최종 보고 문구는 PMO 담당자가 검토합니다. |
```
