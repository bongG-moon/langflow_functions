# HTML 리포트 요소 빠른 참고

Langflow 스티커 메모에 붙여넣기 쉽도록 표를 쓰지 않고 작성한 버전입니다.

보고 싶은 방식에 아래 한글 요소명을 그대로 사용하면 됩니다.

## 사용 가능한 요소

- 리포트 제목 영역: 리포트의 제목, 부제, row 수, 대상 독자, 분석 목적을 상단에 크게 보여줍니다.

- 데이터 범위 요약: 사용한 데이터셋, row 수, preview 여부, 컬럼 수를 카드로 요약합니다.

- 주의사항 박스: 데이터 제한, 오류, preview 경고처럼 사용자가 먼저 알아야 할 내용을 보여줍니다.

- 데이터 없음 안내: 조건에 맞는 데이터가 없을 때 빈 결과 안내를 보여줍니다.

- KPI 카드: 핵심 숫자 지표를 카드 형태로 요약합니다. 예: 총 WIP, 평균 수율, HIGH 건수.

- 증감 카드: 현재값, 기준값, 목표값, 증감률처럼 변화 중심 지표를 카드로 보여줍니다.

- 추이 선 그래프: 날짜나 시간에 따른 숫자 변화 흐름을 선 그래프로 보여줍니다.

- 비교 막대그래프: 공정별, 제품별, 상태별처럼 범주별 숫자 차이를 막대로 비교합니다.

- 묶음 막대그래프: 하나의 범주에서 여러 지표를 나란히 비교합니다. 예: 공정별 WIP와 생산량.

- 누적 구성 막대: 큰 범주 안의 세부 구성을 누적 막대로 보여줍니다. 예: 공정별 상태 구성.

- 도넛 차트: 범주별 비중이나 구성비를 도넛 형태로 보여줍니다.

- 히스토그램: 숫자 컬럼의 분포, 구간별 빈도, 평균, 중앙값, 범위를 보여줍니다.

- 산점도: 두 숫자 지표 사이의 관계, 산포, 추세선, 상관계수를 보여줍니다.

- 교차 히트맵: 두 범주 조합의 값을 색상 강도와 숫자로 보여줍니다.

- 교차표: 두 범주를 행과 열로 놓고 값을 표 형태로 비교합니다.

- 순위 표: 상위 또는 하위 항목을 순위 중심으로 보여줍니다.

- 순위 변동 표: 이전 순위와 현재 순위의 변화가 있을 때 변동을 표로 보여줍니다.

- 상세 데이터 표: 분석에 사용된 row를 사용자가 직접 확인할 수 있도록 표로 보여줍니다.

- 기간 비교 표: 일별, 월별, 주차별 값을 정확한 숫자 표로 비교합니다.

- 이상/예외 표: HIGH, WARN, 임계치 초과 같은 위험 row만 모아 강조해서 보여줍니다.

- 핵심 해석 문장: 차트와 표에서 읽어야 할 핵심 발견사항을 짧은 bullet로 보여줍니다.

- 다음 확인 사항: 후속 확인, 현장 조치, 개선 방향을 짧은 목록으로 제안합니다.

- 생성 기준: 집계 방식, 데이터 기준, 해석 시 주의사항을 하단에 설명합니다.

## 바로 쓸 수 있는 표현

- 핵심 숫자 먼저 보기: 상단에는 KPI 카드 5개를 배치해줘.

- 시간 흐름 보기: DATE별 OUTPUT_QTY 추이 선 그래프를 보여줘.

- 범주 비교하기: PROCESS별 WIP_QTY를 비교 막대그래프로 보여줘.

- 구성비 보기: ALERT_LEVEL별 WIP_QTY 비중을 도넛 차트로 보여줘.

- 분포 보기: YIELD_RATE 분포를 히스토그램으로 보여줘.

- 관계 보기: DEFECT_QTY와 CYCLE_TIME_HR 관계를 산점도로 보여줘.

- 교차 분석하기: PROCESS와 PRODUCT_FAMILY 교차 히트맵을 보여줘.

- 위험 row만 보기: ALERT_LEVEL이 HIGH 또는 WARN인 행만 이상/예외 표로 보여줘.

- 상세 확인하기: 마지막에는 상세 데이터 표를 full width로 보여줘.

- 조치까지 보기: 마지막에는 다음 확인 사항 2개를 제안해줘.

## Langflow 스티커 메모 줄바꿈 팁

- 표 문법은 쓰지 않는 것이 안전합니다. 요소와 설명을 세로 목록으로 작성하는 편이 가장 안정적입니다.

- 일반 Markdown에서는 줄 끝에서 Enter 한 번만 누른 줄바꿈이 화면에서 한 줄로 붙어 보일 수 있습니다.

- 가장 안전한 방법은 항목 사이에 빈 줄을 한 줄씩 넣는 것입니다.

- 그래도 줄바꿈이 붙어 보이면 각 줄 끝에 `<br>`을 넣어보세요.

- 스티커 메모 입력 중 Enter가 저장이나 포커스 이동처럼 동작하면 Shift+Enter로 줄바꿈을 시도해보세요.

- 내용이 길면 하나의 스티커 메모에 모두 넣기보다 "사용 가능한 요소", "바로 쓸 수 있는 표현"처럼 2개 메모로 나누는 것이 보기 좋습니다.
